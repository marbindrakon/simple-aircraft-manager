# Linux desktop build (Flatpak)

Proof-of-concept Flatpak packaging for Simple Aircraft Manager. Produces a
single `.flatpak` bundle file that installs and runs identically to the
macOS `.dmg` and Windows `.exe` builds.

## Prerequisites

A Linux host (or Linux VM — the macOS / Windows builds are produced on
their respective host OSes for the same reason).

```sh
# Distro packages
sudo apt install flatpak flatpak-builder imagemagick   # Debian / Ubuntu
sudo dnf install flatpak flatpak-builder ImageMagick   # Fedora

# Flathub remote (one-time, per user)
flatpak remote-add --user --if-not-exists flathub \
    https://flathub.org/repo/flathub.flatpakrepo

# Runtime + SDK (auto-installed by build-flatpak.sh on first run)
flatpak install --user flathub org.gnome.Platform//47 org.gnome.Sdk//47
```

## Build

From the repo root:

```sh
desktop/flatpak/build-flatpak.sh
# → SimpleAircraftManager-0.1.0-poc.flatpak
```

Override the version with `SAM_BUILD_VERSION=0.2.0 desktop/flatpak/build-flatpak.sh`.

## Install + run

```sh
flatpak install --user SimpleAircraftManager-0.1.0-poc.flatpak
flatpak run app.simpleaircraft.Manager
```

User data (database, config, secret_key, logs) lives at
`~/.var/app/app.simpleaircraft.Manager/data/SimpleAircraftManager/`.

To uninstall and start over:

```sh
flatpak uninstall --user app.simpleaircraft.Manager
rm -rf ~/.var/app/app.simpleaircraft.Manager/
```

## Smoke test

1. **First run**: `flatpak run app.simpleaircraft.Manager` opens a webview
   showing the first-run setup form at `/desktop/setup/`.
2. **No-auth setup**: submit with auth=disabled, no AI provider. Confirm
   `~/.var/app/app.simpleaircraft.Manager/data/SimpleAircraftManager/config.ini`
   exists. Quit, relaunch — the setup redirect no longer fires; aircraft
   list loads.
3. **Persistence**: create an aircraft, add a component, update hours.
   Quit, relaunch, confirm the data is still there.
4. **Auth + keyring**: uninstall, reinstall, redo setup with auth=required
   and an Anthropic API key. Confirm the key round-trips into GNOME Keyring:
   ```sh
   secret-tool search service SimpleAircraftManager
   ```
   If the host doesn't have a SecretService provider running (rare on
   GNOME/KDE), keyring writes will fail with a clear error and the rest of
   the app continues to work — just without AI-import access.
5. **File upload**: in the logbook tab, upload a PDF. The GTK file chooser
   should appear via the FileChooser portal. Confirm the file imports.
6. **Single-instance guard**: launch twice in quick succession. The second
   invocation should show an "already running" dialog rather than racing
   on the loopback port.

## Sandbox permissions

Granted (`finish-args` in the manifest):

| Permission | Why |
|------------|-----|
| `--share=network` | Loopback HTTP between webview and Waitress; outbound HTTPS to Anthropic / OpenAI / Ollama |
| `--share=ipc`, `--socket=wayland`, `--socket=fallback-x11`, `--device=dri` | WebKitGTK rendering |
| `--talk-name=org.freedesktop.secrets` | Keyring access (API keys via `desktop/setup_view.py`) |

Deliberately **not** granted: `--filesystem=`. File uploads from the
webview go through the FileChooser portal automatically.

## Known limitations (POC)

- **Online build only.** The manifest pip-installs from PyPI during build
  (`build-args: --share=network`). For Flathub submission this needs to be
  replaced with a `python3-requirements.json` generated by
  [`flatpak-pip-generator`](https://github.com/flatpak/flatpak-builder-tools/tree/master/pip)
  for offline reproducibility.
- **Bundled, not Flathub.** Single-file bundle distribution; no
  auto-updates. Flathub submission tracked separately.
- **No screenshots in metainfo.xml yet.** Required for Flathub. Add
  `<screenshots>` with publicly-hosted PNG URLs before submitting.
- **`gnome-platform//47` pinned.** Bump the runtime version periodically;
  Flathub policy expects current-or-previous GNOME runtime.

## Files in this directory

| File | Purpose |
|------|---------|
| `app.simpleaircraft.Manager.yml` | flatpak-builder manifest |
| `launcher.sh` | Installed to `/app/bin/simple-aircraft-manager`; sets `PYTHONPATH` + `DJANGO_SETTINGS_MODULE`, execs `python3 -m desktop.launcher` |

| `app.simpleaircraft.Manager.metainfo.xml` | AppStream metadata (used by software centers) |

| `build-flatpak.sh` | Local build helper (mirrors `build-macos.sh` / `build-windows.ps1`) |
